AutoML for Images

1. WorkSpace Setup

AutoML models can be deployed as an endpoint in the user's AzureML workspace. Users can control the compute used for inferencing, use high performance serving with Triton Inference server and can secure the inferencing environment with a virtual network.


2. Compute target setup

You will need to provide a Compute Target that will be used for your AutoML model training. AutoML models for computer vision tasks require GPU SKUs and support NC and ND families. We recommend using the NCsv3-series (with v100 GPUs) for faster training. Using a compute target with a multi-GPU VM SKU will leverage the multiple GPUs to speed up training. Additionally, setting up a compute target with multiple nodes will allow for faster model training by leveraging parallelism, when tuning hyperparameters for your model.


3. Experiment Setup

You can optionally specify the maximum time budget for your AutoML Vison experiment using experiment_timeout_hours - the amount of time in hours before the experiment terminates. If none specified, default experiment timeout is 6 days.


4. dataset operation

If your training data is in a different format (e.g. pascal VOC), you can leverage the helper scripts included with the sample notebooks in this repo to convert the data to JSONL. Once your data is in JSONL format, you can create a labeled dataset


5. Upload jsonl file and images to datastore

Upload the JSONL File and your Training/Validation Dataset to Datastore.


6. auto ml run for image tasks

AutoML for Images supports the following task types: image-classification, image-multi-labeling, image-object-detection, image-instance-segmentation. Select task and then select corresponding models.
